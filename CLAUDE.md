# Wolle Economy Viewer

Streamlit-приложение для расчёта юнит-экономики заказов с маркетплейса Яндекс Маркет.

## Источники данных

- **Excel-отчёты** из Яндекс Маркета (финансовые отчёты, заказы, возвраты)
- **PostgreSQL** — база данных с данными о заказах

## Структура проекта

```
app.py              # точка входа Streamlit
db/
  connection.py     # SQLAlchemy-подключение к PostgreSQL (env-переменные)
parsers/            # парсеры Excel-отчётов Яндекс Маркета
pages/              # дополнительные страницы Streamlit
data/uploads/       # временное хранилище загруженных файлов (в .gitignore)
```

## Команды

```bash
# Запуск приложения
.venv/bin/streamlit run app.py

# Установка зависимостей
.venv/bin/pip install -r requirements.txt
```

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни:

```
DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD
```

## Прогресс

### Сделано
- Инициализирован проект: venv, зависимости, структура директорий
- `app.py` — скелет с загрузкой файлов и кнопкой проверки БД
- `db/connection.py` — подключение через SQLAlchemy + env-переменные

### Предстоит
- Получить структуру Excel-отчётов и схему PostgreSQL от пользователя
- Написать парсеры Excel (`parsers/`)
- Написать запросы к БД (`db/`)
- Реализовать расчёт метрик (`economics.py`)
- Построить таблицы и графики в Streamlit

## Правила разработки

### Общие
- Интерпретатор — `.venv/bin/python`, не системный
- **После каждого значимого шага обновлять раздел «Прогресс»** в этом файле
- Язык UI и комментариев — **русский**; имена переменных/функций — **английский**
- Не коммитить `.env`, загруженные Excel-файлы, содержимое `data/uploads/`

### Код
- Парсеры Excel — в `parsers/`; каждый тип отчёта — отдельный модуль
- Запросы к БД — в `db/`; сырой SQL через `sqlalchemy.text`, никаких ORM-моделей
- Расчёт метрик — в отдельном модуле `economics.py` в корне (чистые функции, без side-эффектов)
- Кэшировать тяжёлые операции через `@st.cache_data`
- Не хранить состояние в глобальных переменных — использовать `st.session_state`

### Streamlit
- `st.set_page_config` вызывается только в `app.py`, не в страницах
- Страницы в `pages/` следуют формату `1_Название.py` (нумерация для порядка)
- Все фильтры — в `st.sidebar`

### Безопасность
- Учётные данные БД только через `.env` / `st.secrets`, никогда не хардкодить
- Входящие файлы валидировать до парсинга (тип, размер)
